Fast Refresh não funciona após mudar para Expo Go e voltar para dev-client.

## Soluções Aplicadas
1. ✅ Removido plugin react-native-reanimated do Babel
2. ✅ Comentado import do react-native-reanimated no index.ts
3. ✅ Simplificado metro.config.js para padrão Expo
4. ✅ Alinhado babel.config.js com o pro/ que funciona
5. ✅ Corrigido uso de children inline no tab.routes.tsx

## Próximo Passo Necessário
**RECONSTRUIR O DEV-CLIENT** - Mudanças no Babel requerem rebuild do app nativo.

## Como Reconstruir

### Opção 1: Via Xcode (Recomendado)
1. Abra o Xcode: `open ios/iwork.xcworkspace`
2. Selecione um simulador disponível
3. Build & Run (Cmd+R)

### Opção 2: Via Terminal (se simulador estiver disponível)
```bash
# Listar simuladores disponíveis
xcrun simctl list devices available

# Reconstruir com simulador específico
npx expo run:ios --device "iPhone 16 Pro"
```

### Opção 3: Limpar e Reconstruir Tudo
```bash
# Limpar tudo
rm -rf node_modules/.cache .expo ios/build android/app/build

# Reinstalar dependências (se necessário)
yarn install

# Reconstruir
npx expo run:ios
```
